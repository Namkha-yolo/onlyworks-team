Tailwind CSS

Utility-class CSS. Already 60% of how the v1 codebase styles things (ow-flex, ow-mb-lg, ow-text-muted). Tailwind is the formalized version, scoped per-component, with no naming collisions.

Why we use this

The current desktop has a 5,700-line single CSS file with class names managed by social convention. The CLAUDE.md rules forbidding .btn-*, .intention-btn-*, .modal-overlay etc. exist because parallel naming systems kept getting reintroduced.

Tailwind eliminates this by removing the naming step entirely. You write styles inline on components: className="flex items-center gap-2 px-4 py-2 bg-purple-600 text-white". No selector to collide with, no file to forget, no dead CSS to clean up. When you delete the component, you delete its styles.

The design tokens (colors, spacing, fonts) live in tailwind.config.ts, so the whole app is themeable from one place — dark mode included.

What you need to know

  • Setup — install, the content array (which files Tailwind scans), the @tailwind directives
  • Spacing scale — p-1 to p-96, gaps, margins. Memorize the small numbers (1-8).
  • Layout — flex, grid, items-*, justify-*, gap-*
  • Sizing — w-*, h-*, min-*, max-*, w-full
  • Typography — text-sm to text-9xl, font-medium, tracking-*, leading-*
  • Colors — the palette, opacity modifiers (bg-purple-600/50)
  • State variants — hover:, focus:, disabled:, dark:, group-hover:, peer-checked:
  • Responsive — sm:, md:, lg:, mobile-first thinking
  • Arbitrary values — w-[42rem], bg-[#abc123] — the escape hatch, use sparingly
  • Custom theme — tailwind.config.ts theme.extend for design tokens
  • Dark mode — darkMode: 'class', toggling via JS
  • @apply escape hatch — rare, but useful for one-off compositions
  • The cn() helper — clsx + tailwind-merge for conditional classes

Patterns you'll see in v2

The cn helper

// lib/utils.ts
import clsx, { type ClassValue } from 'clsx';
import { twMerge } from 'tailwind-merge';

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}

// usage:
<div className={cn(
  'px-4 py-2 rounded',
  variant === 'primary' && 'bg-purple-600 text-white',
  variant === 'ghost' && 'bg-transparent text-purple-600',
  disabled && 'opacity-50 cursor-not-allowed',
)} />

Design tokens in config

// tailwind.config.ts
export default {
  content: ['./src/**/*.{ts,tsx}'],
  darkMode: 'class',
  theme: {
    extend: {
      colors: {
        accent: { DEFAULT: '#7c3aed', light: '#ede9fe', deep: '#6d28d9' },
        paper: '#fafaf9',
        ink: '#080503',
      },
      fontFamily: {
        serif: ['Instrument Serif', 'serif'],
        sans: ['Instrument Sans', 'sans-serif'],
      },
    },
  },
};

Use the design tokens (e.g. bg-accent, font-serif) instead of raw values everywhere. When the brand changes color in 6 months, you edit one file.

Dark mode toggle

// in a wrapper component
<div className={darkMode ? 'dark' : ''}>
  <App className="bg-paper text-ink dark:bg-zinc-900 dark:text-zinc-100" />
</div>

Hands-on: build this

Rebuild the OnlyWorks v1 design-system page in Tailwind:

  1. Open CLAUDE/design-system.html in the desktop repo. Look at the v1 component gallery.
  2. In a Vite + React + Tailwind project, recreate the <Button> component with the 4 variants (primary, secondary, ghost, danger) and 3 sizes (sm, md, lg) using nothing but Tailwind utilities + the cn helper.
  3. Add a dark mode toggle that flips a class on <html> and changes the page colors.
  4. Recreate the <StatCard> component (value, label, icon). It should look identical light and dark.
  5. Use the design tokens from tailwind.config.ts, not hardcoded colors.

How this fits OnlyWorks v2

Tailwind is the styling layer for both the desktop renderer and the website. The website already uses it — look at onlyworks_verified/tailwind.config.ts for the current setup.

v2 ports the v1 utility classes 1:1: ow-flex becomes flex, ow-mb-lg becomes mb-6, ow-text-muted becomes text-muted (via a design token). The 5,700-line styles.css is deleted entirely.
