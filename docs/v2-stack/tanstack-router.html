TanStack Router

Type-safe routing. Replaces v1's showSection() switch statement with real routes, real history, real type-checked link params.

Why we use this

The v1 renderer "routes" by clobbering #main-content.innerHTML via a switch statement in router.js. There's no URL state, no back button, no shareable state. If you want to navigate to a specific report, you call viewReportDetails(reportId) and pray that function exists on window.

TanStack Router gives us:

  • Type-safe links and params. If you typo /reports/$reprotId, the compiler errors.
  • URL state. The user can refresh and end up on the same page. Deep links work.
  • Route-level data loaders. Each route declares its data dependencies; Suspense + TanStack Query orchestrates loading.
  • Code splitting for free. Each route is a dynamic import.
  • Browser-history-aware back button. Even in Electron, where the user's instinct to "go back" works correctly.

The alternative is React Router. We pick TanStack Router for the type safety: it's the only React router where route paths and search params are typed end-to-end.

What you need to know

  • The mental model — file-based vs code-based route definitions (we use code-based for the desktop)
  • createRouter, createRoute, route trees, RouterProvider
  • Path params — $reportId typed via the route definition
  • Search params — typed and validated via Zod schemas
  • <Link> — type-checked to, params, search
  • useNavigate, programmatic navigation
  • Route loaders — running before the route mounts, returning data via Suspense
  • Route context — sharing auth, theme, query client into routes
  • Layouts — parent routes that wrap children (e.g. an app shell with sidebar)
  • Pending / error UI — pendingComponent, errorComponent
  • Code splitting via lazyRouteComponent

Defining routes

// src/routes/__root.tsx
import { createRootRoute, Outlet } from '@tanstack/react-router';

export const rootRoute = createRootRoute({
  component: () => (
    <div className="flex h-screen">
      <Sidebar />
      <main className="flex-1 overflow-auto">
        <Outlet />
      </main>
    </div>
  ),
});

// src/routes/reports/$reportId.tsx
import { createRoute } from '@tanstack/react-router';

export const reportRoute = createRoute({
  getParentRoute: () => rootRoute,
  path: '/reports/$reportId',
  loader: ({ params }) => queryClient.ensureQueryData({
    queryKey: ['report', params.reportId],
    queryFn: () => trpc.reports.get.query({ id: params.reportId }),
  }),
  component: ReportDetail,
});

function ReportDetail() {
  const { reportId } = reportRoute.useParams();
  const { data } = useSuspenseQuery({
    queryKey: ['report', reportId],
    queryFn: () => trpc.reports.get.query({ id: reportId }),
  });
  return <ReportView report={data} />;
}

Search params with Zod validation

const searchSchema = z.object({
  filter: z.enum(['all', 'active', 'completed']).default('all'),
  page: z.number().int().min(1).default(1),
});

export const sessionsRoute = createRoute({
  getParentRoute: () => rootRoute,
  path: '/sessions',
  validateSearch: searchSchema,
  component: SessionsList,
});

function SessionsList() {
  const { filter, page } = sessionsRoute.useSearch();
  // filter and page are typed exactly as the schema says.
}

Hands-on: build this

Build a small typed-routing app on top of your GitHub browser:

  1. Install @tanstack/react-router and the devtools.
  2. Three routes: / (search form), /u/$username (user's repos), /u/$username/r/$repoName (one repo's detail).
  3. Use validateSearch on the user route to accept a typed ?sort=stars|updated.
  4. Use Link with typed params throughout. Deliberately typo a path and watch the compile error.
  5. Add a route loader that pre-fetches the data via TanStack Query before mounting.
  6. Add a parent layout route that renders a back button + breadcrumbs.

How this fits OnlyWorks v2

The desktop has ~12 top-level routes: dashboard, sessions, activity, reports, report detail, session detail, settings, OW profile, manager, etc. v2 represents each one as a real route. Sidebar nav becomes typed <Link>s. The viewReportDetails(id) pattern becomes navigate({ to: '/reports/$reportId', params: { reportId: id } }).

For deep linking (opening the app at a specific report), the Electron onlyworks:// protocol handler just turns the URL into a router navigation. No more "if route is /report, load it via window.viewReportDetails" hacks.
