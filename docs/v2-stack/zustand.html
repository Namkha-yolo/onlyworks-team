08 · State & data

Zustand

UI state without the Redux ceremony. One hook, no providers, no actions, no reducers — and ~1KB.

Why we use this

v1 has a hand-rolled window.AppStore + ~6 global flags (isCapturing, currentSessionId, sessionStartTime, currentSection, etc.) updated from various modules. It's invisible state, untyped, untestable.

v2 uses two state systems, by category:

  • Server state (anything that lives on the backend) — TanStack Query
  • UI state (anything that doesn't outlive a session) — Zustand

"UI state" means: theme, sidebar collapsed/expanded, currently-open modal, the active session's local capture state, draft form contents, layout preferences. Things React state hooks could handle if they didn't have to be lifted six levels.

We avoid Redux for the same reason React's docs do: too much boilerplate per change. Zustand is what people reach for in 2024+.

What you need to know

  • create — the one factory you'll call
  • The store as a hook — useStore(s => s.theme)
  • Selectors — pass a function, only re-render when that slice changes
  • Actions — just functions on the store, no dispatching
  • Equality functions — shallow for picking multiple fields, useShallow
  • Middleware — persist (localStorage), devtools, immer
  • Slicing — one big store with sub-slices, or several small stores; both work, pick a convention
  • Computed values — derive in the selector, don't store derived state
  • Testing — the store is just a function call; no provider mocking
  • Outside React — you can read/write the store from non-component code (e.g. an IPC handler)

Patterns you'll see in v2

A simple typed store

// src/state/ui-store.ts
import { create } from 'zustand';

interface UIState {
  sidebarCollapsed: boolean;
  theme: 'light' | 'dark' | 'auto';

  toggleSidebar: () => void;
  setTheme: (t: 'light' | 'dark' | 'auto') => void;
}

export const useUI = create<UIState>((set) => ({
  sidebarCollapsed: false,
  theme: 'auto',

  toggleSidebar: () => set((s) => ({ sidebarCollapsed: !s.sidebarCollapsed })),
  setTheme: (theme) => set({ theme }),
}));

Persist middleware (localStorage)

import { persist } from 'zustand/middleware';

export const useUI = create<UIState>()(
  persist(
    (set) => ({ /* ... */ }),
    { name: 'ow-ui' }, // localStorage key
  ),
);

Selectors prevent re-renders

// Re-renders on ANY UI state change — bad
const ui = useUI();

// Re-renders only when theme changes — good
const theme = useUI((s) => s.theme);

// Multiple fields — use useShallow
import { useShallow } from 'zustand/react/shallow';
const { theme, sidebarCollapsed } = useUI(
  useShallow((s) => ({ theme: s.theme, sidebarCollapsed: s.sidebarCollapsed }))
);

Reading the store from outside React

// e.g. inside an IPC handler that needs the current theme
const theme = useUI.getState().theme;

// or subscribe to changes:
const unsub = useUI.subscribe((s) => s.theme, (theme) => {
  console.log('theme changed to', theme);
});

Hands-on: build this

Add a Zustand store to your GitHub browser project:

  1. Install Zustand. Create src/state/ui.ts.
  2. Track three things in UI state: theme ('light' | 'dark'), recentlyViewedUsernames (string[]), favoriteRepoIds (number[]).
  3. Use the persist middleware for theme and favorites; don't persist recentlyViewed (it should reset between sessions).
  4. Build a theme switcher in the sidebar. Verify selector subscriptions: when you click theme, only components that read theme re-render.
  5. Use useShallow to subscribe to two fields at once.
  6. From a non-React place (a util function), read the current theme via getState().

How this fits OnlyWorks v2

v2 has 2–3 Zustand stores: one for UI (theme, sidebar, modals), one for session state (the local mirror of the running capture session), maybe one for tutorial / onboarding flow. Together they replace the entire window.AppStore + scattered globals from v1.

Server-state (reports, sessions list, OW Profile) all goes through TanStack Query. If you find yourself reaching for Zustand to cache a fetched API response, stop — that's TanStack Query's job.
