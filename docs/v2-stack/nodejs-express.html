Node.js + Express

The backend server. JWT-authenticated REST + tRPC, talks to Supabase Postgres and Gemini AI. Hosted on Render.

Why we use this

The backend has to: handle OAuth, manage sessions, take batched OCR frames from the desktop and invoke Gemini, generate reports, expose hiring search, send transactional email, run the talent-community magic-link flow, and serve the public OW profile pages. Express + Node is the obvious default: every contributor knows it, every library targets it, every host runs it.

The current backend (Express 4) is at /Users/xerneas/onlyworks-backend-server. It uses MVC (controllers / services / repositories), JWT middleware, Helmet for security headers, an in-memory rate limiter, and Supabase as the only database. v2 evolves this rather than rewriting it.

  • Logging — structured JSON, levels, request log middleware
  • Testing with Vitest — supertest-style integration tests

import express from 'express';
import cors from 'cors';
import helmet from 'helmet';
import compression from 'compression';

const app = express();

app.use(helmet({ contentSecurityPolicy: { /* ... */ } }));
app.use(cors({
  origin: (origin, cb) => {
    if (!origin) return cb(null, true);                   // Electron has no Origin
    if (allowList.includes(origin)) return cb(null, true);
    cb(new Error('Not allowed by CORS'));
  },
  credentials: true,

app.use(express.json({ limit: '10mb' }));

app.use(notFoundHandler);

Auth middleware

import jwt from 'jsonwebtoken';

export function authenticateUser(req, res, next) {
  const auth = req.headers.authorization;
  if (!auth?.startsWith('Bearer ')) return next(new ApiError('AUTH_REQUIRED'));

  try {
    const decoded = jwt.verify(auth.slice(7), process.env.JWT_SECRET);
    req.user = { userId: decoded.userId, email: decoded.email };
    next();
  } catch {
    next(new ApiError('AUTH_INVALID'));
  }
}

A typed route with Zod

import { z } from 'zod';

const createReportSchema = z.object({
  startDate: z.string().date(),
  endDate: z.string().date(),
  title: z.string().min(1).max(120).optional(),
});

router.post('/generate-range', authenticateUser, async (req, res) => {
  const input = createReportSchema.parse(req.body);
  const report = await reportService.generateRange(req.user.userId, input);
  res.json({ success: true, data: report });
});

Background worker with single-flight guard

let running = false;
setInterval(async () => {
  if (running) return;
  running = true;
  try {
    const result = await staleSessionService.finalize(STALE_THRESHOLD_MIN);
    if (result.processed > 0) logger.info('Stale finalizer', result);
  } catch (e) {
    logger.error('Stale finalizer failed', { err: e.message });
  } finally {
    running = false;
  }
}, 10 * 60 * 1000);

process.on('SIGTERM', () => { /* cleanup */ });

Hands-on: build this

Build a tiny replica of the v1 backend's auth slice:

  1. New Express app with TypeScript, Zod, jsonwebtoken, helmet, cors, compression.
  2. Routes: POST /auth/register (email + password), POST /auth/login (issue JWT), POST /auth/refresh (rotate refresh token), GET /me (auth-required, returns user info).
  3. Store users in memory (Map keyed by email) for now — Supabase comes next chapter.
  4. Per-IP rate limit on /auth/login (5/min). On /me, use the normal limiter.
  5. Middleware that validates the JWT and populates req.user.
  6. Add a global error handler that returns { success: false, error } JSON.
  7. Write a Vitest integration test using supertest that hits each endpoint.

How this fits OnlyWorks v2

v2 keeps Express on the backend. The MVC layout (routes → controllers → services → repositories) stays. What changes:

  • TypeScript across the whole backend.
  • Zod parsing on every request body / query / params.
  • tRPC alongside REST — the website's typed client and the desktop both consume tRPC procedures backed by the same services. REST endpoints stay for share-link viewers and webhook callbacks where the consumer isn't ours.
  • The 10-min stale-session finalizer stays exactly as-is. Same with SIGTERM handling and SSE cleanup.
  • Vitest for tests, run on CI per PR.

Read /Users/xerneas/onlyworks-backend-server/src/app.js and pick one route (e.g. src/routes/authRoutes.js). That's the reference for how every backend route is shaped in v2.
