Python + PyTorch

The foundation of everything ML. We train our own model on top of PyTorch — but first you need to be able to read a training loop without panicking.

Why we use this

The endgame for OnlyWorks AI: train a proprietary multimodal model that takes a screenshot + activity context (OCR text, app, window title) and emits the same kind of interpretation we currently get from Gemini Vision. Replacing Gemini at scale is the bet that makes the product economics work past 100k users.

Every modern model lives in the PyTorch ecosystem. The model weights are PyTorch tensors. The training loop is a PyTorch nn.Module + optimizer.step() dance. Hugging Face Transformers is a PyTorch library. LoRA and QLoRA fine-tuning are PyTorch operations. If you're going to do anything with our model, you read and write PyTorch.

You don't need to be a research-paper-publishing PyTorch wizard. You need to:

  • Read a training script and understand what each line does.
  • Modify a training config (learning rate, batch size, optimizer).
  • Spot when something is wrong (loss is NaN, gradient explodes, GPU is OOM).
  • Run inference, save and load checkpoints, evaluate on a held-out set.

Patterns you'll see in our model training code

A minimal PyTorch training loop

import torch
from torch import nn, optim
from torch.utils.data import DataLoader

model = MyModel().to('cuda')
optimizer = optim.AdamW(model.parameters(), lr=3e-4, weight_decay=0.01)
loss_fn = nn.CrossEntropyLoss()
loader = DataLoader(my_dataset, batch_size=32, shuffle=True, num_workers=4)

for epoch in range(num_epochs):
    model.train()
    for batch in loader:
        x, y = batch['input'].to('cuda'), batch['label'].to('cuda')

        logits = model(x)
        loss = loss_fn(logits, y)

        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

    # Eval
    model.eval()
    with torch.no_grad():
        for batch in val_loader:

            pass

    torch.save(model.state_dict(), f'ckpt_epoch_{epoch}.pt')

A custom Dataset class

from torch.utils.data import Dataset
from PIL import Image

class TrainingFrameDataset(Dataset):
    def __init__(self, manifest_path, transform):
        self.records = json.load(open(manifest_path))
        self.transform = transform

    def __len__(self):
        return len(self.records)

    def __getitem__(self, idx):
        record = self.records[idx]
        image = Image.open(record['image_path']).convert('RGB')
        image = self.transform(image)
        return {
            'image': image,
            'text': record['ocr_text'],
            'label': record['activity_label'],
        }

Mixed precision (free speedup on GPU)

scaler = torch.cuda.amp.GradScaler()

for batch in loader:
    with torch.cuda.amp.autocast():
        logits = model(batch['x'])
        loss = loss_fn(logits, batch['y'])

    optimizer.zero_grad()
    scaler.scale(loss).backward()
    scaler.step(optimizer)
    scaler.update()

Hands-on: build this

Train MNIST classification end-to-end as a warm-up:

  1. Install Python 3.11 via uv. Create a venv.
  2. Install torch, torchvision, matplotlib.
  3. Load MNIST via torchvision.datasets.MNIST.
  4. Define a small CNN (2 conv + 1 dense) as an nn.Module.
  5. Train for 5 epochs. Plot the loss curve. Confirm test accuracy > 98%.
  6. Save the model. Load it in a separate script. Run inference on a single image.
  7. If you have a GPU: move to it (.to('cuda')) and add mixed precision. Time both.
  8. Add Weights & Biases logging (we'll cover this in the MLOps doc).

Goal: you should be able to recite the training loop from memory after this exercise. Karpathy's "Zero to Hero" videos prepare you for it; this exercise solidifies it.

How this fits OnlyWorks v2

The OnlyWorks model-training stack is Python first. The Node-based training-data-service in the desktop captures full-res frames + metadata and stores them in training_frames table + Supabase Storage. The Python side reads from that storage, builds datasets, trains models.

Once you've done the MNIST exercise, the next steps in this track are: Hugging Face Transformers (load existing models), Vision-Language Models (the architecture we're starting from), and LoRA fine-tuning (how we adapt a base model to our data).

Cost note

Local CPU training is fine for the MNIST warm-up. The minute you move to anything bigger (a fine-tune of a 7B model), you need a GPU. We rent them on Modal / RunPod / Lambda Labs. Ask before you spin one up. See training infrastructure.
