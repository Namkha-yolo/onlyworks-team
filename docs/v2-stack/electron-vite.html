electron-vite

Vite, configured for Electron's three contexts (main, preload, renderer) with hot reload across all of them. The day-to-day dev experience.

Why we use this

Electron has three TypeScript contexts: the main process (Node), the preload (a sandboxed Node-lite), and the renderer (browser). Each needs different bundle targets, different externals, different injected globals. Setting Vite up for all three by hand is a weekend; electron-vite is a thin wrapper that does it for you.

The win: edit src/renderer/... and the React app hot-reloads in the open Electron window in <100ms. Edit src/main/... and the main process restarts cleanly, keeping the renderer up. No more "kill the app, restart, log back in, navigate to the page" loop.

The alternatives (electron-forge, hand-rolled vite-plugin-electron setups) are fine but electron-vite is the lowest-friction modern choice.

What you need to know

  • Project structure — src/main/, src/preload/, src/renderer/
  • One electron.vite.config.ts — with separate main, preload, renderer Vite configs
  • HMR — renderer gets standard Vite HMR; main/preload get fast restart
  • External Node modules — how to mark uiohook-napi, sharp, etc. as externals so they're loaded from node_modules at runtime
  • Env vars — MAIN_VITE_, PRELOAD_VITE_, RENDERER_VITE_ prefixes for each context
  • Building — electron-vite build outputs to out/{main,preload,renderer}; then electron-builder packages it
  • The boilerplate template — npm create @quick-start/electron gives a clean starting point

electron.vite.config.ts (typical)

import { defineConfig, externalizeDepsPlugin } from 'electron-vite';
import react from '@vitejs/plugin-react';
import path from 'node:path';

export default defineConfig({
  main: {
    plugins: [externalizeDepsPlugin()],
    resolve: {
      alias: { '@shared': path.resolve('src/shared') },
    },
  },
  preload: {
    plugins: [externalizeDepsPlugin()],
  },
  renderer: {
    plugins: [react()],
    resolve: {
      alias: {
        '@': path.resolve('src/renderer/src'),
        '@shared': path.resolve('src/shared'),
      },
    },
  },
});

Hands-on: build this

Scaffold a v2-shaped Electron + Vite + React + TS project from scratch:

  1. Run npm create @quick-start/electron@latest electron-lab -- --template react-ts.
  2. Run pnpm install and pnpm dev. Confirm HMR works on the renderer.
  3. Add a path alias @shared pointing to src/shared. Put a Zod schema there. Import it in both main and renderer; confirm both compile.
  4. Add a native module dependency (e.g. sharp). Mark it as an external and confirm it loads at runtime.
  5. Add a renderer env var RENDERER_VITE_BACKEND_URL. Read it via import.meta.env. Confirm it's not visible in main.
  6. Run pnpm build. Inspect the out/ directory. Run electron . on it directly and confirm the bundled app starts.

How this fits OnlyWorks v2

This is the dev loop you'll live in. Every time you edit a React component, you'll see it update in the running Electron window. Every time you edit a main-process service, the process restarts and the renderer reconnects. Every time you edit a Zod schema in src/shared, both sides recompile.

Once we ship, this same toolchain feeds the production build — electron-vite build produces the out/ bundles that electron-builder turns into DMG and NSIS installers.
