macOS Vision OCR

A compiled Swift binary that calls Apple's Vision framework. ~50ms per screenshot, free, offline, accurate. Replaces tesseract.js on the platform where it matters most.

Why we use this

tesseract.js (the WASM port of Tesseract) works everywhere but is slow (200–800ms per screenshot) and accuracy is mediocre on screen content with mixed fonts. Apple's Vision framework, built into every macOS, runs ~5x faster and produces noticeably better text on UI screenshots.

We ship a tiny compiled Swift binary alongside the app. The Node main process spawns it with a screenshot path, reads the extracted text from stdout. No JS bindings, no FFI, no rebuild matrix.

The OCR text feeds Pipeline 2 (activity analysis): every 3 seconds we extract on-screen text, batch it with app/window-title metadata, and send to Gemini Flash for activity interpretation. Quality of OCR directly affects quality of reports.

What you need to know

  • Apple's Vision framework — VNRecognizeTextRequest, recognition levels (fast vs accurate), language settings
  • Swift basics — enough to read src/native/ocr_helper and modify it
  • Compiling a Swift binary — swiftc from Command Line Tools, no Xcode project needed
  • The build script — src/native/build-ocr-helper.sh in v1 — runs at prebuild:mac time
  • Bundling the binary — via build.mac.extraResources in package.json, the binary lands in process.resourcesPath/ocr_helper
  • Calling from Node — child_process.exec with the binary path, timeout, max buffer
  • Fallback chain — compiled binary → Swift interpreter (CLT installed?) → tesseract.js
  • Universal binary — compile for x86_64 + arm64 with lipo

The Swift helper (skeleton)

// ocr_helper.swift
import Foundation
import Vision
import AppKit

guard CommandLine.arguments.count > 1 else {
  print("ERROR: No image path provided")
  exit(1)
}

let imagePath = CommandLine.arguments[1]
guard let image = NSImage(contentsOfFile: imagePath),
      let cgImage = image.cgImage(forProposedRect: nil, context: nil, hints: nil) else {
  print("ERROR: Could not load image")
  exit(1)
}

let request = VNRecognizeTextRequest { request, error in
  guard let observations = request.results as? [VNRecognizedTextObservation] else { return }
  for obs in observations {
    if let top = obs.topCandidates(1).first {
      print(top.string)
    }
  }
}
request.recognitionLevel = .fast
request.usesLanguageCorrection = false

let handler = VNImageRequestHandler(cgImage: cgImage)
try handler.perform([request])

The build script

# src/native/build-ocr-helper.sh
#!/bin/bash
set -e
swiftc -O \
  -target arm64-apple-macos11.0 \
  src/native/ocr_helper.swift \
  -o src/native/ocr_helper_arm64

swiftc -O \
  -target x86_64-apple-macos11.0 \
  src/native/ocr_helper.swift \
  -o src/native/ocr_helper_x86_64

lipo -create -output src/native/ocr_helper \
  src/native/ocr_helper_arm64 \
  src/native/ocr_helper_x86_64

Calling from Node

// src/main/services/ocr/macos.ts
import { exec } from 'node:child_process';
import path from 'node:path';

export async function extractTextMacOS(imagePath: string): Promise<string> {
  const binary = app.isPackaged
    ? path.join(process.resourcesPath, 'ocr_helper')
    : path.join(__dirname, '..', '..', 'native', 'ocr_helper');

  return new Promise((resolve, reject) => {
    exec(`"${binary}" "${imagePath}"`,
      { timeout: 10_000, maxBuffer: 1024 * 1024 },
      (err, stdout, stderr) => {
        if (err) return reject(new Error(stderr || err.message));
        const out = stdout.trim();
        if (out.startsWith('ERROR:')) return reject(new Error(out));
        resolve(out);
      });
  });
}

Bundling the binary

// package.json
{
  "scripts": {
    "prebuild:mac": "bash src/native/build-ocr-helper.sh"
  },
  "build": {
    "mac": {
      "extraResources": [
        { "from": "src/native/ocr_helper", "to": "ocr_helper" }
      ]
    }
  }
}

Hands-on: build this

Compile and ship the v1 OCR helper standalone:

  1. Copy v1's src/native/ocr_helper.swift + build script to your Electron lab project.
  2. Run bash build-ocr-helper.sh. Confirm you get a universal binary at ./ocr_helper.
  3. From the command line: ./ocr_helper /path/to/screenshot.png — confirm you see extracted text.
  4. Wire it into the main process behind a tRPC procedure ocr.extract(imagePath: string).
  5. Add a renderer button that takes a screenshot via desktopCapturer, writes to a temp file, and shows the extracted text.
  6. Add extraResources + asarUnpack entries. Run pnpm dist:mac. Confirm OCR still works in the packaged app.
  7. Edit the Swift file: change recognitionLevel from .fast to .accurate. Time the difference.

How this fits OnlyWorks v2

The macOS path in v2's OcrConsumer is: compiled binary first, Swift interpreter second, tesseract.js third. Compiled binary should hit on every shipped app (we always ship it bundled). Swift interpreter only triggers on dev machines where the bundled binary is missing. Tesseract is the global fallback when neither macOS path works.

Cost in time and bytes: the universal binary is ~3 MB, takes seconds to compile, runs ~50ms per screenshot, free per call. Compared to Gemini Vision at ~$0.0005 per call and 1-2s latency, this is a >100x cost reduction on the activity-pipeline OCR step.

Privacy note

OCR runs locally; the screenshot never leaves the user's machine. Only the extracted text (after the sensitive-content filter passes) gets sent to the backend. This is load-bearing for the privacy story — don't change it.
