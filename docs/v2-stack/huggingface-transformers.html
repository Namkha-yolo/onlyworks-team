Hugging Face Transformers

The library + hub for every open-source model that matters. Download a model in one line, fine-tune it in fifty, deploy it from the hub. The center of gravity for our model training stack.

Why we use this

Hugging Face is to ML what npm is to JS. The transformers library lets you load any model from the Hub with AutoModel.from_pretrained('org/model-name') and start using it in three lines of code. The Hub hosts hundreds of thousands of pretrained models, datasets, and demos. The datasets library handles loading + preprocessing at scale. The tokenizers library handles fast text encoding. The PEFT library adds LoRA / QLoRA on top.

For OnlyWorks: we start from an open Vision-Language Model (Qwen-VL, LLaVA, or similar), fine-tune it on our screenshot + activity dataset, and push the result to a private Hub repo. From there we serve it via vLLM or a similar inference framework. Every step is Hugging Face native.

What you need to know

Core libraries

  • transformers — the model + tokenizer + pipeline abstraction
  • datasets — load + map + filter + split datasets
  • tokenizers — fast Rust-based tokenization
  • accelerate — one-call multi-GPU + mixed precision + device placement
  • peft — LoRA + QLoRA + adapter management
  • bitsandbytes — quantization for memory savings
  • safetensors — safer model weight format (default now)

Auto classes

  • AutoModel, AutoModelForCausalLM, AutoModelForVisualQuestionAnswering
  • AutoTokenizer, AutoProcessor (for multimodal)
  • AutoConfig — inspect a model without loading weights

Training

  • Trainer — high-level trainer with logging, evaluation, checkpointing
  • TrainingArguments — the config object (learning rate, batch size, gradient accumulation, mixed precision, warmup, …)
  • Custom loops — when Trainer isn't flexible enough
  • Data collators — batching + padding for variable-length inputs

Inference

  • pipeline() — one-liner inference for common tasks
  • Generation — model.generate(...), temperature, top-p, beam search
  • Streaming — TextIteratorStreamer for token-by-token output

The Hub

  • Model cards — read before you use a model
  • Licenses — Apache 2.0, MIT, CreativeML, custom commercial restrictions
  • huggingface-cli login + access tokens, private vs public repos
  • Spaces — hosted demos, useful for sharing experiments

Patterns you'll see in v2

Loading a model + tokenizer

from transformers import AutoTokenizer, AutoModelForCausalLM

model_id = 'Qwen/Qwen2.5-7B-Instruct'
tokenizer = AutoTokenizer.from_pretrained(model_id)
model = AutoModelForCausalLM.from_pretrained(model_id, torch_dtype='auto', device_map='auto')

inputs = tokenizer('Summarize this work session...', return_tensors='pt').to(model.device)
outputs = model.generate(**inputs, max_new_tokens=200, do_sample=True, temperature=0.7)
print(tokenizer.decode(outputs[0], skip_special_tokens=True))

Loading a dataset

from datasets import load_dataset

# from the Hub
dataset = load_dataset('squad', split='train')

# from local files
dataset = load_dataset('json', data_files='train.jsonl', split='train')

# preprocess
def tokenize(example):
    return tokenizer(example['text'], truncation=True, max_length=512)

dataset = dataset.map(tokenize, batched=True, remove_columns=dataset.column_names)

Trainer setup (minimal)

from transformers import Trainer, TrainingArguments, DataCollatorForLanguageModeling

args = TrainingArguments(
    output_dir='./ckpts',
    learning_rate=2e-5,
    per_device_train_batch_size=4,
    gradient_accumulation_steps=8,
    num_train_epochs=3,
    logging_steps=50,
    save_steps=500,
    bf16=True,
    report_to='wandb',
)

trainer = Trainer(
    model=model,
    args=args,
    train_dataset=tokenized_train,
    eval_dataset=tokenized_val,
    data_collator=DataCollatorForLanguageModeling(tokenizer, mlm=False),
)

trainer.train()

Push to the Hub

trainer.push_to_hub(
    'onlyworks/activity-interpreter-v0.1',
    private=True,
)

Hands-on: build this

Fine-tune a small model on a public dataset:

  1. Pick a small base model (e.g. distilbert-base-uncased).
  2. Pick a public sentiment dataset (e.g. sst2 or imdb) via datasets.
  3. Use the Trainer to fine-tune for 1–2 epochs on a sample.
  4. Compute accuracy on the test split. Confirm >80%.
  5. Save with trainer.save_model(). Load it into a separate inference script.
  6. Run pipeline('sentiment-analysis', model='./model_dir') and classify three custom sentences.
  7. Push to the Hub as a private repo. Pull it back from a fresh notebook to confirm it works.

This is doable on a Mac (CPU) in about an hour with a small model. Anything bigger needs a GPU.
