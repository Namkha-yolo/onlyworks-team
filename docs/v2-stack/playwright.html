Playwright

End-to-end browser + Electron testing. Drives a real browser, asserts on real DOM, takes real screenshots. Catches the regressions that unit tests can't.

Why we use this

Unit tests catch logic regressions. They don't catch "the sidebar nav broke after the React 19 upgrade", "the OAuth flow loses the redirect URL on Safari", or "after sign-in, the dashboard route 404s on the first load."

Playwright drives a real Chrome / Firefox / WebKit instance, runs through a flow as a user would, and asserts on the outcome. It also has first-class support for Electron via _electron — launch the packaged app, attach to its windows, run the same kinds of assertions.

For the website (Next.js), Playwright tests the marketing flows, the OAuth round-trip with Supabase, the Talent Community magic-link signup. For the desktop, Playwright tests the onboarding flow, sign-in, session start/end, and report visibility — the critical paths that have broken before.

What you need to know

  • Test structure — test('description', async ({ page }) => ...)
  • Locators — getByRole, getByLabel, getByText, getByTestId
  • Why selector strategy matters — prefer roles + labels over CSS selectors
  • Auto-waiting — Playwright auto-waits for elements; you almost never sleep
  • Assertions — expect(locator).toBeVisible(), toHaveText, toHaveURL
  • Multiple browsers — run the same test across Chromium, Firefox, WebKit
  • Trace viewer — on failure, watch the test as a movie; inspect the DOM at any frame
  • Fixtures — share auth state, build a logged-in user once, reuse across tests
  • Network interception — mock backend responses, simulate failures
  • Electron testing — _electron.launch, attaching to windows, IPC inspection
  • CI integration — the official GitHub Action, cross-browser matrices, retries

Patterns you'll see in v2

A web flow test

// e2e/talent-community.spec.ts
import { test, expect } from '@playwright/test';

test('user can request a magic link', async ({ page }) => {
  await page.goto('/talent');
  await page.getByLabel('Email').fill('test@example.com');
  await page.getByLabel('Full name').fill('Test User');
  await page.getByRole('button', { name: /join/i }).click();
  await expect(page.getByText(/check your inbox/i)).toBeVisible();
});

Auth fixture (reusable logged-in state)

// tests/setup/auth.ts
import { test as base } from '@playwright/test';

export const test = base.extend({
  authedPage: async ({ page }, use) => {
    await page.goto('/login');
    await page.getByLabel('Email').fill(process.env.E2E_EMAIL!);
    await page.getByLabel('Password').fill(process.env.E2E_PASSWORD!);
    await page.getByRole('button', { name: 'Sign in' }).click();
    await page.waitForURL('/dashboard');
    await use(page);
  },
});

Electron test

import { _electron as electron, expect, test } from '@playwright/test';

test('desktop boots to the auth screen', async () => {
  const app = await electron.launch({ args: ['.'] });
  const window = await app.firstWindow();
  await expect(window.getByText('Sign in')).toBeVisible();
  await app.close();
});

Network mock

test('shows error state when backend is down', async ({ page }) => {
  await page.route('**/api/reports/user', (route) => route.fulfill({
    status: 500,
    body: JSON.stringify({ success: false, error: 'oops' }),
  }));
  await page.goto('/dashboard');
  await expect(page.getByText(/couldn.t load reports/i)).toBeVisible();
});

Hands-on: build this

Add Playwright to your Next.js verified-lab from the previous chapter:

  1. pnpm create playwright — pick TS, default settings.
  2. Write 3 tests: homepage loads, profile page loads with a real GitHub username, profile page 404s with a nonsense username.
  3. Use getByRole + getByText as primary locators — not CSS classes.
  4. Add a network mock test for the 500-error case.
  5. Run pnpm exec playwright test --ui and explore the UI runner + trace viewer.
  6. Wire into GitHub Actions: install Playwright browsers, run tests, upload the HTML report as an artifact.
  7. Bonus: an Electron test for your Electron lab project — just launch the app and assert that a window opens.

How this fits OnlyWorks v2

Playwright covers the v2 critical paths that broke in v1:

  • Onboarding (welcome → permissions → profile-setup → dashboard)
  • Sign-in (system browser OAuth + the polling rendezvous with the desktop)
  • Start session → end session → daily report visible
  • Generate a custom report from selected sessions
  • Share report via link / email
  • Talent Community magic-link signup → OW Profile public view
  • Hiring search (anonymous teaser → authed full view)

One Playwright test per critical path. Run them on every PR. Re-run on a nightly schedule against production-like data. The cost is small; the prevention of "we shipped a broken sign-in flow" regressions is large.
