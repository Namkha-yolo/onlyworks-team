Learn the v2 stack.

Everything you need to read and watch before you write a line of production code on OnlyWorks v2. Pick a track, follow the order, build the mini-projects.

How to use these docs

Each page covers one piece of the v2 stack: why we use it, what to learn, the best official docs and YouTube videos to start with, deeper resources for when you want to go further, and a hands-on mini-project to prove to yourself that you actually understood it.

You will not learn everything at once. The goal is to read enough that you can open a v2 PR, follow what's going on, and ask better questions. Treat this as a reference you come back to, not a one-time slog.

What you'll be doing in v2

OnlyWorks v2 is a rebuild of the Electron desktop app. Read docs/03-architecture-review.html for the strategic rationale. The short version:

  • Throw away the renderer. 37 <script> tags, 5,700 lines of CSS, innerHTML string templating, manually-mirrored window.* exports. Replace with React + TypeScript + Vite + Tailwind + shadcn/ui.
  • Keep the native plumbing. Capture loop, OCR helpers (Swift binary on macOS, PowerShell wrapper on Windows), auto-updater, OAuth + token refresh. Years of platform bug fixes embedded; we port, we don't rewrite.
  • Restructure the capture pipeline. One unified CaptureEngine emitting screenshot events to multiple consumers (OCR, training data, metrics) instead of two parallel 1,400-line god classes.
  • Replace the 169-channel IPC bridge with tRPC over IPC so renderer↔main is typed end-to-end. The handlers that are just HTTP pass-throughs fold into direct fetch calls from the renderer.
  • Train our own multimodal model on the screenshot + activity data flowing through the training pipeline. Goal: replace Gemini at scale once we have enough labeled frames.

House rules

  • Default to the official docs. Tutorials and YouTube are best for getting the shape of a concept; the docs are where you go to be correct.
  • One thing at a time. Don't try to learn TypeScript + React + Vite + Tailwind in the same week. Pick one, build something tiny, ship it, then pick the next.
  • Build the mini-project on every page. If you skip it, you didn't learn the thing — you just watched videos.
  • Ask before you spend. GPU rentals, paid Hugging Face plans, W&B teams — ping the founders before you charge anything.
  • Privacy is load-bearing. The desktop captures screen content. Every line of code that touches a screenshot is governed by our sensitive-content filter and the user's consent settings. When in doubt, ask.
