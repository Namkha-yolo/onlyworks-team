electron-builder + updater

DMG, NSIS, code signing, Apple notarization, and silent auto-updates from GitHub releases. The part of the stack that's nobody's favorite but always works.

Why we use this

Once electron-vite has produced out/main, out/preload, out/renderer, you still need to:

  • Package those into a single app bundle
  • Code-sign for macOS Gatekeeper
  • Notarize with Apple (or users see a scary dialog)
  • Sign for Windows SmartScreen
  • Generate .dmg and .exe installers
  • Build for x64 + arm64 on macOS, x64 + portable on Windows
  • Push to GitHub Releases with the right latest-mac.yml / latest.yml manifests
  • Trigger silent in-app updates via electron-updater

electron-builder + electron-updater handle every step. v1's config in package.json already works — v2 ports it nearly verbatim. This isn't a place to experiment.

What you need to know

  • The build block in package.json — the config lives here
  • App ID (com.onlyworks.desktop) and product name
  • Targets per platform — dmg+zip on mac, nsis+portable on win
  • Artifact naming — dots, not spaces (GitHub URL-encodes spaces, breaks auto-update URLs)
  • asarUnpack — native modules that can't live in the app.asar archive (uiohook-napi, sharp, active-win, tesseract.js-core, PowerShell scripts)
  • Code signing on macOS — Apple Developer ID, entitlements, hardened runtime
  • Notarization — the afterSign hook + @electron/notarize
  • Windows signing — EV cert, signAndEditExecutable, verifyUpdateCodeSignature
  • The publish config — provider: 'github', the release repo
  • electron-updater — autoUpdater.checkForUpdatesAndNotify(), update events, the startup-screen UX
  • Protocols — the onlyworks:// custom URL scheme registration
  • Extra resources — how to ship the compiled OCR binary alongside the app

Patterns you'll see in v2

The build config (simplified from v1)

// package.json
{
  "build": {
    "appId": "com.onlyworks.desktop",
    "productName": "OnlyWorks Desktop",
    "afterSign": "build/notarize.js",
    "files": ["out/**/*", "package.json"],
    "asarUnpack": [
      "node_modules/uiohook-napi/**",
      "node_modules/sharp/**",
      "node_modules/active-win/**"
    ],
    "publish": [{
      "provider": "github",
      "owner": "Namkha-yolo",
      "repo": "ONLYWORKS_DIST"
    }],
    "mac": {
      "category": "public.app-category.productivity",
      "hardenedRuntime": true,
      "gatekeeperAssess": false,
      "entitlements": "build/entitlements.mac.plist",
      "target": [
        { "target": "dmg", "arch": ["x64", "arm64"] },
        { "target": "zip", "arch": ["x64", "arm64"] }
      ],
      "artifactName": "OnlyWorks.Desktop-${version}-${arch}.${ext}"
    },
    "win": {
      "target": [
        { "target": "nsis", "arch": ["x64"] },
        { "target": "portable", "arch": ["x64"] }
      ]
    }
  }
}

The auto-updater (wire-up)

// src/main/auto-updater.ts
import { autoUpdater } from 'electron-updater';

autoUpdater.on('update-available', (info) => {
  sendToRenderer('update-status', { state: 'available', version: info.version });
});

autoUpdater.on('update-downloaded', () => {
  sendToRenderer('update-status', { state: 'downloaded' });
});

autoUpdater.checkForUpdatesAndNotify();

Notarization (the afterSign hook)

// build/notarize.js
const { notarize } = require('@electron/notarize');

exports.default = async function (context) {
  if (context.electronPlatformName !== 'darwin') return;

  await notarize({
    appBundleId: 'com.onlyworks.desktop',
    appPath: `${context.appOutDir}/OnlyWorks Desktop.app`,
    appleId: process.env.APPLE_ID,
    appleIdPassword: process.env.APPLE_APP_SPECIFIC_PASSWORD,
    teamId: process.env.APPLE_TEAM_ID,
  });
};

Hands-on: build this

Take your electron-vite lab project from the previous chapter and ship a release:

  1. Install electron-builder and electron-updater.
  2. Add a build block with appId, productName, mac DMG target, win NSIS target.
  3. Run pnpm dist:mac (or win). Inspect the generated dist/.
  4. Drag the DMG to /Applications and run the app. Confirm it works.
  5. Wire up autoUpdater.checkForUpdatesAndNotify() in main.
  6. Set publish.provider: 'github' pointing at a private dist repo of yours.
  7. Publish 1.0.0, then bump to 1.0.1, publish again, and verify the running 1.0.0 receives the update.
  8. (macOS-only) Get signing + notarization working with your Apple Developer ID. The first time is hours; every time after is minutes.
Ask first

Apple Developer Program ($99/yr) and a Windows EV code-signing certificate are real money. Don't buy either yourself — the team already has both. Ask before touching the production signing keys.

How this fits OnlyWorks v2

v1's build block is solid — we port it. The asarUnpack list, the entitlements, the notarization hook, the artifact name format, the Mica option on Windows — all of it carries over with minimal change. The main updates are pointing at v2 source paths (out/main/main.js instead of src/main.js) and dropping any v1-specific paths.

The auto-update flow in v1 (src/services/auto-updater.js + src/core/auto-updater-flow.js) is well-thought and ports almost verbatim. The startup screen with "skip update / install now" prompts is a real-user pain killer — don't simplify it away.
