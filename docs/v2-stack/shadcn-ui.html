shadcn/ui

Not a component library. A collection of copy-and-paste components built on Radix primitives + Tailwind. You own the code; you customize freely.

Why we use this

Component libraries (Material UI, Chakra, Mantine) lock you into their design system. When you need to deviate — "make this dropdown look slightly different on Tuesdays" — you fight the library. shadcn/ui solves this by being not a library at all: it's a CLI that copies source files into your repo. After that, the code is yours.

The underlying components are built on Radix UI — unstyled, accessible primitives with proper keyboard navigation, focus management, ARIA roles, and edge cases (e.g. closing a dropdown when you click outside) all handled correctly. shadcn/ui wraps Radix with sensible Tailwind defaults.

For OnlyWorks, this means we get a properly accessible modal, dropdown, dialog, command palette, tooltip — all in our own files, all styled to match the v2 brand — without writing them from scratch and without depending on a library.

What you need to know

  • The mental model — "components live in your repo" vs "components live in node_modules"
  • CLI — npx shadcn@latest init, then add components one at a time
  • Setup — the components.json file, path aliases, the cn() helper
  • What gets added — check in the generated files; they're yours to edit
  • Radix primitives underneath — understanding why Dialog, DropdownMenu, Popover behave correctly
  • Compound components — <Dialog><DialogTrigger><DialogContent> patterns
  • Theming — CSS variables in globals.css, the :root + .dark pattern, customizing via tailwind.config.ts
  • Customizing components — the components are in your repo. Just edit them.
  • Available components — Button, Input, Dialog, DropdownMenu, Select, Toast, Form (paired with React Hook Form), Command, Table, Tabs, Tooltip, …
  • When to write your own — shadcn covers 80%; the OnlyWorks-specific stuff (StatCard, SessionPanel, ReportCard) you build yourself

Adding a component

npx shadcn@latest add button dialog dropdown-menu input
# files appear in src/components/ui/

A typical compound component

import { Dialog, DialogContent, DialogHeader, DialogTitle, DialogTrigger } from '@/components/ui/dialog';
import { Button } from '@/components/ui/button';

function ConfirmDelete({ onConfirm }: { onConfirm: () => void }) {
  return (
    <Dialog>
      <DialogTrigger asChild>
        <Button variant="destructive">Delete report</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>Delete this report?</DialogTitle>
        </DialogHeader>
        <p>This cannot be undone.</p>
        <Button variant="destructive" onClick={onConfirm}>Confirm</Button>
      </DialogContent>
    </Dialog>
  );
}

Theming via CSS variables

/* globals.css */
:root {
  --background: 60 9% 98%;
  --foreground: 24 10% 4%;
  --primary: 263 70% 50%;       /* purple */
  --primary-foreground: 0 0% 100%;
  /* ... */
}

.dark {
  --background: 24 10% 4%;
  --foreground: 60 9% 98%;
  /* ... */
}

shadcn uses HSL CSS vars so you can adjust the whole theme by editing two blocks. Don't hardcode colors in component files; use the tokens.

Hands-on: build this

Set up shadcn/ui in your Vite + React + Tailwind project from the previous chapter:

  1. npx shadcn@latest init — pick the default theme.
  2. Add: button, input, dialog, dropdown-menu, toast.
  3. Build a "Settings" mini-page with a name input, a save button, a confirm-delete dialog, and a profile dropdown menu.
  4. On submit, show a success toast.
  5. Add a dark mode toggle that flips the theme by toggling a class on <html>.
  6. Open one of the generated component files (e.g. components/ui/button.tsx). Change the default variant's color. Confirm the change propagates to every <Button> in your app.

Goal: feel the difference between "use a library" and "own the code". The component is yours; touch it.

How this fits OnlyWorks v2

The desktop renderer uses shadcn primitives for everything that's not OnlyWorks-specific: dialogs, dropdowns, command palette, toasts, tables, tabs. The OnlyWorks-specific components (<SessionPanel>, <DailyBriefing>, <OWProfileCard>) are built in the same style — copy the shadcn patterns and adapt.

The website at onlyworks_verified already uses shadcn; see components.json and components/ui/ there.
