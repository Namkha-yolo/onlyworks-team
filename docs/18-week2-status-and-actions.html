src/main/trpc/capture.ts:

import { z } from 'zod'
import { router, publicProcedure } from './trpc'

// Tracks whether a capture session is active. In-memory for W2; the real
// capture engine lands later and will own this.
let capturing = false

export const captureRouter = router({
  // MUTATION — logs in MAIN, returns the ApiResult envelope
  start: publicProcedure
    .input(z.object({ reason: z.string().optional() }).optional())
    .mutation(({ input }) => {
      capturing = true
      // This console.log runs in the MAIN process => shows in the pnpm dev terminal.
      console.log('[capture.start] capture started', input ?? {})
      return { ok: true as const, data: { started: true } }
    }),

  // QUERY — returns { ok: true, data: { running } }
  status: publicProcedure.query(() => ({ ok: true as const, data: { running: capturing } }))
})

src/main/trpc/router.ts:

import { router } from './trpc'
import { captureRouter } from './capture'

export const appRouter = router({
  capture: captureRouter
})

// The renderer imports ONLY this type (never the runtime router).
export type AppRouter = typeof appRouter

Why as const on ok: it makes the return type { ok: true; data: ... } not { ok: boolean; data: ... }, so callers get a precise type. status returns the same envelope — { ok: true, data: { running } }, not a bare boolean — so every procedure speaks the shared ApiResult shape. Why an optional input on start: A's button currently calls with no args, so an optional input keeps the call ergonomic while still validating if something is passed.

Wire the IPC transport by hand: ONE ipcMain.handle channel in main that drives a tRPC caller

The hand-rolled transport is one IPC channel. The renderer sends { path, type, input }; main looks the procedure up via a tRPC caller and returns the result. Using createCallerFactory (not raw router internals) means zod validation and the procedure resolvers run exactly as tRPC intends — no any, no hand-parsing of the router tree.

src/main/trpc/ipc.ts:

import { ipcMain } from 'electron'
import { createCallerFactory } from './trpc'
import { appRouter } from './router'

// Single, well-known channel name. Keep it in one place so preload + main agree.
export const TRPC_IPC_CHANNEL = 'trpc' as const

export interface TrpcIpcRequest {
  path: string // e.g. 'capture.start'
  type: 'query' | 'mutation'
  input: unknown
}

export type TrpcIpcResponse =
  | { ok: true; result: unknown }
  | { ok: false; error: { message: string; code?: string } }

const createCaller = createCallerFactory(appRouter)

export function registerTrpcIpc(): void {
  ipcMain.handle(
    TRPC_IPC_CHANNEL,
    async (_event, req: TrpcIpcRequest): Promise<TrpcIpcResponse> => {
      try {
        // Empty context for now; later this is where session/auth goes.
        const caller = createCaller({})
        // Walk the dotted path to the procedure fn on the caller.
        const segments = req.path.split('.')
        let target: unknown = caller
        for (const seg of segments) {
          target = (target as Record<string, unknown>)[seg]
        }
        if (typeof target !== 'function') {
          throw new Error(`Unknown procedure: ${req.path}`)
        }
        const result = await (target as (input: unknown) => Promise<unknown>)(req.input)
        return { ok: true, result }
      } catch (err) {
        const message = err instanceof Error ? err.message : 'Unknown tRPC IPC error'
        return { ok: false, error: { message } }
      }
    }
  )
}

Renderer — a typed client that imports ONLY the type. This is where the typing magic happens: inferRouterInputs / inferRouterOutputs from @trpc/server turn AppRouter into per-procedure input/output types, so a typo fails to compile.

The renderer's tsconfig.web.json does NOT include src/main, but a type-only import type still resolves across the boundary because the file physically exists and @trpc/server is installed. Use a relative type-only import (the @/ alias only covers src/renderer/src, so use a relative ../../../main/... path).

src/renderer/src/lib/trpc.ts:

import type { inferRouterInputs, inferRouterOutputs } from '@trpc/server'
import type { AppRouter } from '../../../main/trpc/router'

type RouterInputs = inferRouterInputs<AppRouter>
type RouterOutputs = inferRouterOutputs<AppRouter>

async function call<T>(path: string, type: 'query' | 'mutation', input: unknown): Promise<T> {
  const res = await window.trpc.request({ path, type, input })
  if (!res.ok) throw new Error(res.error.message)
  return res.result as T
}

// Typed facade. Add one line per procedure — see docs/HOW-TO-ADD-PROCEDURE.md.
export const trpc = {
  capture: {
    start: (input?: RouterInputs['capture']['start']) =>
      call<RouterOutputs['capture']['start']>('capture.start', 'mutation', input),
    status: () =>
      call<RouterOutputs['capture']['status']>('capture.status', 'query', undefined)
  }
}

Because the return types are derived from AppRouter, trpc.capture.start() resolves to { ok: true; data: { started: boolean } } and trpc.capture.status() to { ok: true; data: { running: boolean } } — fully typed, no any. This src/renderer/src/lib/trpc.ts is the single shared renderer client: Groups A and C both import it from @/lib/trpc rather than writing their own. C adds its auth.* methods to this same file (one plain line each, per the HOW-TO-ADD-PROCEDURE doc below).

Set the BrowserWindow security flags EXPLICITLY and explain why

In src/main/index.ts, make the security posture explicit in webPreferences (currently it only sets sandbox: false):

const mainWindow = new BrowserWindow({
  width: 1100,
  height: 720,
  show: false,
  autoHideMenuBar: true,
  webPreferences: {
    preload: join(__dirname, '../preload/index.js'),
    contextIsolation: true,   // renderer JS cannot reach preload/Node internals
    nodeIntegration: false,   // no Node APIs (require/process) in the renderer
    sandbox: false            // keep: preload uses @electron-toolkit/preload (Node)
  }
})

Why each one:

  • contextIsolation: true — the renderer's window and the preload run in separate JS contexts. The ONLY things the renderer can touch are what you explicitly contextBridge.exposeInMainWorld. This is what makes exposing just window.trpc.request safe and prevents the renderer from grabbing ipcRenderer. (true is the Electron default since v12, but the decision doc and our security review want it stated, not assumed.)
  • nodeIntegration: false — the renderer cannot require('fs') or touch process. All privileged work happens in main, reached only through your one IPC channel.
  • sandbox: false — left as-is. The preload imports @electron-toolkit/preload/@electron-toolkit/utils, which use Node; a full sandbox would block that. sandbox: false does NOT undo contextIsolation — they're independent. Note this trade-off in the PR's "Why" section so the security reviewer sees it was deliberate.

Do NOT flip contextIsolation to false to "make types/imports work" — if something won't import, the import chain is wrong (fix that), not the security flag.

Write docs/HOW-TO-ADD-PROCEDURE.md (1 page)

Create docs/HOW-TO-ADD-PROCEDURE.md so A and C can add procedures without reading the transport code:

# How to add a tRPC procedure

The transport is hand-rolled tRPC v11 over one Electron IPC channel
(`src/main/trpc/ipc.ts`). You almost never touch the transport — you add a
procedure to a router and one typed line in the renderer facade.

## 1. Add the procedure (main)
Edit the relevant router in `src/main/trpc/` (e.g. `capture.ts`):
```ts
rename: publicProcedure
  .input(z.object({ id: z.string(), name: z.string() }))
  .mutation(({ input }) => {
    // runs in MAIN — console.log shows in the pnpm dev terminal
    return { ok: true as const }
  })
```
Use `.query(...)` for reads, `.mutation(...)` for writes. Always validate
input with `zod`. If it's a new router, create it and mount it in
`src/main/trpc/router.ts` under `appRouter`.

## 2. Add one typed line (renderer)
In `src/renderer/src/lib/trpc.ts`, add a method that mirrors the path:
```ts
rename: (input: RouterInputs['capture']['rename']) =>
  call<RouterOutputs['capture']['rename']>('capture.rename', 'mutation', input),
```
The input/output types come from `AppRouter`, so a typo'd field fails to compile.

## 3. Call it
```ts
import { trpc } from '@/lib/trpc'
await trpc.capture.rename({ id: 'a', name: 'b' })
```

## Rules
- Never expose `ipcRenderer` on `window` — only `window.trpc.request`.
- Never import the runtime router into the renderer — `import type { AppRouter }` only.
- No `any` / `as unknown as`. If types won't line up, the import chain is wrong.

Watch out for

  • Raw ipcRenderer leaking to the renderer: NEVER do contextBridge.exposeInMainWorld('ipcRenderer', ipcRenderer) or put it on window. Expose only the narrow trpc.request function. If a teammate needs a new capability, add a procedure, not a new exposed Electron primitive.
  • Turning off contextIsolation to 'make it work': if the renderer can't see window.trpc or a type won't import, the bug is in your exposeInMainWorld name / d.ts / import path — fix that. Flipping contextIsolation:false defeats the entire security boundary and will be rejected in review.
  • Using any or as unknown as to force the renderer types: that's the symptom of a broken import chain. The renderer must import type { AppRouter } (type-only) from src/main/trpc/router and derive types via inferRouterInputs/Outputs. If you reach for as unknown as, your AppRouter import is wrong.
  • Putting @trpc/server / zod in devDependencies: electron.vite.config.ts uses externalizeDepsPlugin() for main+preload, which externalizes dependencies and bundles nothing from devDependencies into the main bundle — main would fail to resolve @trpc/server at runtime. They MUST be in dependencies.
  • Unpinned deps: install with -E and verify package.json shows 11.6.0 / 3.25.76 with no ^/~. The PR checklist explicitly requires exact-pinned new deps.
  • Importing the runtime appRouter (value) into the renderer instead of the type: that drags Node-only tRPC server code into the browser bundle. Use import type exclusively; the renderer only ever sends strings over IPC.
  • Registering ipcMain.handle after createWindow or twice: register once, before the window loads, or early renderer calls race the handler / you get 'second handler for trpc' errors on re-register.

import type {
  CaptureStartOutput,
  CaptureStatusOutput
} from '@shared/contracts'
import { isApiSuccess } from '@shared/types/api-envelope'
import { trpc } from '@/lib/trpc'

/** Thin renderer-side wrapper over Lihan's typed tRPC-over-IPC client. */
export async function startCapture(): Promise<{ started: boolean }> {
  const res: CaptureStartOutput = await trpc.capture.start()
  if (!isApiSuccess(res)) {
    // Throw on the envelope error so TanStack Query routes it to onError.
    throw new Error(res.message)
  }
  return res.data
}

export async function getCaptureStatus(): Promise<{ running: boolean }> {
  const res: CaptureStatusOutput = await trpc.capture.status()
  if (!isApiSuccess(res)) throw new Error(res.message)
  return res.data
}

Decode the JWT's exp in the main process — no library needed:

// src/main/services/jwt.ts
export function accessTokenExpiryMs(accessToken: string): number {
  const payload = accessToken.split('.')[1]                 // header.PAYLOAD.signature
  if (!payload) return 0
  const json = Buffer.from(payload, 'base64url').toString('utf8')
  const { exp } = JSON.parse(json) as { exp?: number }      // exp = seconds since epoch
  return exp ? exp * 1000 : 0                               // -> ms, comparable to Date.now()
}

4. Encrypted storage (electron-store + safeStorage) — still required

safeStorage encrypts the token bytes with a key kept in the OS keychain. Never write a raw key or raw token to disk — that's a hard stop.

// src/main/services/token-store.ts
import { safeStorage } from 'electron'
import Store from 'electron-store'
import type { AuthTokens } from '../types/auth'

const store = new Store<{ authBlob?: string; email?: string }>({ name: 'config' })

export function saveTokens(tokens: AuthTokens): void {
  if (!safeStorage.isEncryptionAvailable()) throw new Error('OS secure storage unavailable')
  store.set('authBlob', safeStorage.encryptString(JSON.stringify(tokens)).toString('base64'))
}
export function loadTokens(): AuthTokens | null {
  const blob = store.get('authBlob')
  if (!blob || !safeStorage.isEncryptionAvailable()) return null
  try { return JSON.parse(safeStorage.decryptString(Buffer.from(blob, 'base64'))) as AuthTokens }
  catch { clearTokens(); return null }
}
export function saveEmail(email: string): void { store.set('email', email) }
export function loadEmail(): string | undefined { return store.get('email') }
export function clearTokens(): void { store.delete('authBlob'); store.delete('email') }

5. backend-auth service — call the real endpoints

// src/main/services/backend-auth.ts
import { saveTokens, loadTokens, clearTokens, saveEmail, loadEmail } from './token-store'
import { accessTokenExpiryMs } from './jwt'
import type { AuthStatus } from '../types/auth'

const BASE = process.env.OW_BACKEND_URL ?? '' // staging base URL from Namkha (config, not hardcoded)

async function post(path: string, body: unknown): Promise<any> {
  const res = await fetch(`${BASE}${path}`, {
    method: 'POST', headers: { 'content-type': 'application/json' }, body: JSON.stringify(body)
  })
  const json = await res.json()
  // auth endpoints are rate-limited (5/min on /email, 10/min on /email/verify); a 429 returns `error` as a STRING, not an object
  if (!json.success) throw new Error(typeof json.error === 'string' ? json.error : (json.error?.message ?? `HTTP ${res.status}`))
  return json.data
}

export async function requestEmailCode(email: string): Promise<{ resend_after_seconds: number }> {
  return post('/api/auth/email', { email })            // emails the 6-digit code
}
export async function verifyEmailCode(email: string, code: string): Promise<AuthStatus> {
  const data = await post('/api/auth/email/verify', { email, code }) // { tokens, user, is_new_user }
  saveTokens(data.tokens); saveEmail(data.user.email)
  return { signedIn: true, email: data.user.email }
}
async function refresh(): Promise<boolean> {
  const t = loadTokens(); if (!t) return false
  try { const data = await post('/api/auth/refresh', { refresh_token: t.refresh_token })
        saveTokens(data.tokens ?? data); return true }
  catch { clearTokens(); return false }
}
const SKEW_MS = 60_000
export async function getValidAccessToken(): Promise<string | null> {
  const t = loadTokens(); if (!t) return null
  if (Date.now() >= accessTokenExpiryMs(t.access_token) - SKEW_MS) {
    if (!(await refresh())) return null
  }
  return loadTokens()?.access_token ?? null
}
export async function status(): Promise<AuthStatus> {
  const tok = await getValidAccessToken()
  return tok ? { signedIn: true, email: loadEmail() } : { signedIn: false }
}
export function signOut(): void { clearTokens() }

async function unwrap<T>(p: Promise<ApiResult<T>>): Promise<T> {
  const r = await p
  if (!isApiSuccess(r)) throw new Error(r.message)
  return r.data
}
